Landmark Coding Patterns

  • All content should be contained in a landmark:
    • banner (top level)
    • complementary
    • contentinfo (top level)
    • main (top level)
    • navigation
    • region (sub sections and unique content)
    • search
  • Must use labels when a landmark of the same type is used more than once
  • Should not nest landmarks of the same type
  • From a design stand point having too few or two many landmarks on a page will reduce the utility of landmarks to people using screen readers. A rule of thumb is to have between 4-10 landmarks on a page, although more complex pages may require more landmarks.
